title: Sui SDK v2 Beta
---

This guide covers the NAVI SDK v2 beta packages for Sui SDK 2.x consumers.

## Runtime

- Use Node.js 22.x or newer.
- Use ESM imports. The v2 beta packages do not expose CommonJS `require` exports.
- Install the v2 beta packages independently from the legacy v1 line.

```bash
npm install @mysten/sui@^2
npm install @naviprotocol/lending@<published-beta-tag-or-version> @naviprotocol/wallet-client@<published-beta-tag-or-version>
npm install @naviprotocol/astros-aggregator-sdk@<published-beta-tag-or-version> @naviprotocol/astros-bridge-sdk@<published-beta-tag-or-version> @naviprotocol/astros-dca-sdk@<published-beta-tag-or-version>
```

Before beta packages are published to npm, install NAVI-provided tarballs or
preview packages generated from the SDK v2 build. Do not use `@next` or `@2`
unless that npm dist-tag/version has been published.

## Client Setup

```ts
import { SuiJsonRpcClient, getJsonRpcFullnodeUrl } from '@mysten/sui/jsonRpc'

const client = new SuiJsonRpcClient({
  network: 'mainnet',
  url: getJsonRpcFullnodeUrl('mainnet')
})
```

Read methods accept a Sui SDK v2 client contract. JSON-RPC is a compatibility transport; do not pass old `@mysten/sui.js` clients or v1 `TransactionBlock` values.

## Lending Read Path

```ts
import { getCoins, getLendingState } from '@naviprotocol/lending'
```

```ts
await depositCoinPTB(tx, '0x2::sui::SUI', coin, {

```

## Wallet Client

```ts
import { WalletClient, WatchSigner } from '@naviprotocol/wallet-client'
import { getJsonRpcFullnodeUrl } from '@mysten/sui/jsonRpc'

const signer = new WatchSigner(address)
const walletClient = new WalletClient({
  signer,
  client: {
    url: getJsonRpcFullnodeUrl('mainnet')
  }
})

const dryRun = await walletClient.balance.sendCoin('0x2::sui::SUI', address, 1_000_000, {
  dryRun: true
})

dryRun.kind
dryRun.effects?.status?.status
```

For real execution, pass an authorized Sui SDK v2 signer from the connected wallet. Wallet methods return NAVI result DTOs such as `NaviWalletTransactionResult`; raw JSON-RPC response types are not part of the stable v2 contract.

```ts
import { buildSwapPTBFromQuote, getQuote } from '@naviprotocol/astros-aggregator-sdk'
import { Transaction } from '@mysten/sui/transactions'

const tx = new Transaction()

  '0xa99b8952d4f7d947ea77fe0ecdcc9e5fc0bcab2841d6e2a5aa00c3044e5544b5::navx::NAVX',
  1_000_000
)

const coinOut = await buildSwapPTBFromQuote(address, tx, 990_000, coinIn, quote, 0, false)

tx.transferObjects([coinOut], address)
```

## Bridge

The v2 beta bridge package keeps the Mayan provider internal and lazy-loaded. Root imports do not load Mayan or Sui v1. For Sui-source bridge execution, the SDK builds provider-specific transaction bytes internally, converts them to a Sui SDK v2 `Transaction`, calls the `signTransaction` function supplied by the app wallet, executes the signed transaction with the supplied v2 `SuiJsonRpcClient`, and returns a `BridgeSwapTransaction`.

The business flow is intentionally the same as the v1 bridge package: quote,
Mayan route builder, wallet signature, source-chain execution, and status
polling. The v2 package changes the integration boundary, not the route or
amount semantics. Apps should not install Mayan or Sui v1, should not parse
Mayan's internal transaction bytes, and should not pass v1 `TransactionBlock`
values to wallet code.

## Frontend And External Integration

Use the v2 beta packages as ESM-only packages on Node.js 22 or newer. Do not install or pass `@mysten/sui.js` clients into v2 SDK calls.

For bridge integration:

```ts
import {
  getQuote,
  getTransaction,
  searchSupportTokens,
  swap,
  type WalletConnection
} from '@naviprotocol/astros-bridge-sdk'
import { SuiJsonRpcClient, getJsonRpcFullnodeUrl } from '@mysten/sui/jsonRpc'

const client = new SuiJsonRpcClient({
  network: 'mainnet',
  url: getJsonRpcFullnodeUrl('mainnet')
})

const fromToken = (await searchSupportTokens(1999, suiUsdcCoinType))[0]
const toToken = (await searchSupportTokens(42161, arbitrumUsdcAddress))[0]
if (!fromToken || !toToken) throw new Error('Required bridge tokens are not available')

const quote = await getQuote(fromToken, toToken, '0.6', { slippageBps: 50 })
if (!quote.routes.length) throw new Error('No bridge route is available')

const walletConnection: WalletConnection = {
  sui: {
    provider: client,
    signTransaction: async ({ transaction }) => {
      return connectedWallet.signTransaction({ transaction })
    }
  }
}

const tx = await swap(quote.routes[0], suiAddress, evmRecipientAddress, walletConnection)
const latest = await getTransaction(tx.id)

if (latest.bridgeStatus === 'completed') {
  latest.bridgeDestTxHash
}
```

The app owns wallet connection, user approval, account selection, and UI state. The SDK owns quote/status requests, lazy Mayan loading, Sui-source transaction conversion, source-chain execution, and the returned bridge DTO. For real transactions, use an explicitly authorized wallet and small test amounts until your integration has passed status polling on the target route.

For EVM-source gasless Mayan routes, the SDK may return a Mayan order hash
instead of an EVM transaction hash. Treat the returned `BridgeSwapTransaction`
as the source for status polling instead of assuming every id is waitable through
the source-chain wallet provider.

## Migration Rules

- Do not import `@mysten/sui.js`.
- Do not expose v1 `SuiClient`, `TransactionBlock`, or raw v1 response types in application-level contracts.
- Keep third-party Sui v1 SDKs out of root app and SDK entry points. If a v1 adapter is unavoidable, load it lazily and keep it internal.
- Build, typecheck, and test v2 packages with Node.js 22.x.
