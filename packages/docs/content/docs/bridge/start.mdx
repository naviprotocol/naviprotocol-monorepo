## Install SDK

```bash
npm install @naviprotocol/astros-bridge-sdk@<published-beta-tag-or-version> @mysten/sui@^2
```

## Configure API Key

```ts
import { config } from '@naviprotocol/astros-bridge-sdk'

config({
  apiKey: 'your api key'
})
```

If your integration is already provisioned through NAVI backend defaults, this
step can be skipped. Use `config({ apiKey })` when your environment requires an
explicit bridge API key.

## Query Supported Chains

```ts
import { getSupportChains } from '@naviprotocol/astros-bridge-sdk'

const SUI_BRIDGE_CHAIN_ID = 1999
const ARBITRUM_CHAIN_ID = 42161

const supportChains = await getSupportChains()

const suiChain = supportChains.find((chain) => chain.id === SUI_BRIDGE_CHAIN_ID)
const arbitrumChain = supportChains.find((chain) => chain.id === ARBITRUM_CHAIN_ID)

if (!suiChain || !arbitrumChain) {
  throw new Error('Required bridge chains are not available')
}
```

`1999` is the Bridge/Mayan chain id for Sui.
`0` is the Bridge/Mayan chain id for Solana.

## Query Bridge Tokens

```ts
import { searchSupportTokens } from '@naviprotocol/astros-bridge-sdk'

const [suiUsdc] = await searchSupportTokens(
  1999,
  '0xdba34672e30cb065b1f93e3ab55318768fd6fef66c15942c9f7cb846e2f900e7::usdc::USDC'
)

const [arbitrumUsdc] = await searchSupportTokens(
  42161,
  '0xaf88d065e77c8cc2239327c5edb3a432268e5831'
)

if (!suiUsdc || !arbitrumUsdc) {
  throw new Error('Required bridge tokens are not available')
}
```

## Get Quote

```ts
import { getQuote } from '@naviprotocol/astros-bridge-sdk'

const quotes = await getQuote(suiUsdc, arbitrumUsdc, '0.6', {
  slippageBps: 50
})

if (!quotes.routes.length) {
  throw new Error('No bridge route is available')
}
```

## Execute A Sui-Source Bridge

For Sui-source routes, the SDK keeps the Mayan provider internal and lazy-loaded.
The app provides a Sui SDK v2 gRPC/Core client (such as `SuiGrpcClient`) and a
wallet-standard `signTransaction` function. The SDK builds the provider
transaction internally, converts it to a v2 `Transaction`, requests the wallet
signature, executes the source-chain transaction, and returns a
`BridgeSwapTransaction`.

```ts
import { swap, type WalletConnection } from '@naviprotocol/astros-bridge-sdk'
import { GrpcWebFetchTransport, SuiGrpcClient } from '@mysten/sui/grpc'

const client = new SuiGrpcClient({
  network: 'mainnet',
  transport: new GrpcWebFetchTransport({
    baseUrl: 'https://fullnode.mainnet.sui.io:443',
    format: 'binary'
  })
})

const walletConnection: WalletConnection = {
  sui: {
    provider: client,
    signTransaction: async ({ transaction }) => {
      return connectedWallet.signTransaction({ transaction })
    }
  }
}

const transaction = await swap(quotes.routes[0], suiAddress, evmRecipientAddress, walletConnection)
```

<Callout>
  If you need to stay on the legacy JSON-RPC transport temporarily, use
  `SuiJsonRpcClient` from `@mysten/sui/jsonRpc` as the `provider` instead. JSON-RPC
  is deprecated and scheduled for removal on 2026-07-31; new integrations should
  use gRPC/Core clients like `SuiGrpcClient` directly.
</Callout>

The v2 SDK keeps the same business path as the v1 bridge package: quote,
Mayan route builder, wallet signature, source-chain execution, and status
polling. The migration only changes the public SDK boundary. Apps pass a Sui SDK
v2 client and v2 wallet signer; the SDK owns the internal Mayan adapter and the
v1-to-v2 transaction conversion.

Do not pass v1 `TransactionBlock` values to the app wallet, and do not parse
Mayan's internal Sui bytes in application code. If source-chain execution fails,
`swap` rejects instead of returning a processing transaction.

## Query Transaction Status

```ts
import { getTransaction } from '@naviprotocol/astros-bridge-sdk'

const latestTransaction = await getTransaction(transaction.id)

switch (latestTransaction.bridgeStatus) {
  case 'processing':
    break
  case 'completed':
    latestTransaction.bridgeDestTxHash
    break
  case 'fail':
    latestTransaction.bridgeRefundTxHash
    break
}
```

Do not import Mayan or Sui v1 SDKs in the application. They are implementation
details of the bridge package's lazy execution path.
